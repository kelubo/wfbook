# 云原生架构

## 概述

云原生架构是一种构建和运行应用的方法，利用云计算的弹性和分布式特性，实现高可用、可扩展、可维护的系统设计。

---

## 一、云原生核心概念

### 1.1 云原生定义

云原生应用具有以下特征：
- **容器化**：应用及其依赖被打包成容器
- **微服务**：应用被拆分成独立的微服务
- **持续交付**：自动化的构建、测试和部署流程
- **DevOps**：开发和运维团队紧密协作
- **弹性伸缩**：根据负载自动调整资源
- **服务网格**：微服务间的流量管理和安全

### 1.2 云原生架构原则

```
┌─────────────────────────────────────────────────────────┐
│                    云原生架构                           │
├─────────────────────────────────────────────────────────┤
│  1. 模块化设计        2. 声明式API       3. 可观测性     │
│  4. 故障隔离          5. 自动恢复        6. 渐进式交付   │
│  7. 安全左移          8. 基础设施即代码   9. 自动化      │
└─────────────────────────────────────────────────────────┘
```

## 二、Kubernetes最佳实践

### 2.1 Pod设计模式

#### Sidecar模式

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: app-with-sidecar
spec:
  containers:
  - name: main-app
    image: myapp:latest
    ports:
    - containerPort: 8080
    volumeMounts:
    - name: shared-data
      mountPath: /data
  - name: sidecar
    image: sidecar:latest
    volumeMounts:
    - name: shared-data
      mountPath: /data
  volumes:
  - name: shared-data
    emptyDir: {}
```

#### Init Container模式

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: app-with-init
spec:
  initContainers:
  - name: init-db
    image: busybox
    command: ['sh', '-c', 'until nslookup db; do echo waiting for db; sleep 2; done;']
  - name: init-config
    image: busybox
    command: ['sh', '-c', 'wget -O /config/config.yaml http://config-server/config']
    volumeMounts:
    - name: config
      mountPath: /config
  containers:
  - name: main-app
    image: myapp:latest
    volumeMounts:
    - name: config
      mountPath: /config
  volumes:
  - name: config
    emptyDir: {}
```

### 2.2 Deployment配置

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: sensor-service
  labels:
    app: sensor-service
spec:
  replicas: 3
  selector:
    matchLabels:
      app: sensor-service
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    metadata:
      labels:
        app: sensor-service
      annotations:
        prometheus.io/scrape: "true"
        prometheus.io/port: "8080"
    spec:
      containers:
      - name: sensor-service
        image: sensor-service:1.0.0
        ports:
        - containerPort: 8080
        resources:
          requests:
            cpu: "100m"
            memory: "128Mi"
          limits:
            cpu: "500m"
            memory: "256Mi"
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /ready
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 5
        env:
        - name: DB_HOST
          valueFrom:
            configMapKeyRef:
              name: sensor-config
              key: db_host
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: sensor-secret
              key: db_password
```

## 三、服务网格

### 3.1 Istio配置

```yaml
# 安装Istio
istioctl install --set profile=demo -y

# 启用自动注入
kubectl label namespace default istio-injection=enabled

# VirtualService配置
apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: sensor-vs
spec:
  hosts:
  - sensor-service
  http:
  - route:
    - destination:
        host: sensor-service
        subset: v1
      weight: 90
    - destination:
        host: sensor-service
        subset: v2
      weight: 10

# DestinationRule配置
apiVersion: networking.istio.io/v1alpha3
kind: DestinationRule
metadata:
  name: sensor-dr
spec:
  host: sensor-service
  subsets:
  - name: v1
    labels:
      version: v1
  - name: v2
    labels:
      version: v2
  trafficPolicy:
    loadBalancer:
      simple: ROUND_ROBIN
```

### 3.2 服务网格流量管理

```yaml
# 超时配置
apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: sensor-vs
spec:
  hosts:
  - sensor-service
  http:
  - route:
    - destination:
        host: sensor-service
    timeout: 5s
    retries:
      attempts: 3
      perTryTimeout: 2s

# 熔断配置
apiVersion: networking.istio.io/v1alpha3
kind: DestinationRule
metadata:
  name: sensor-dr
spec:
  host: sensor-service
  trafficPolicy:
    connectionPool:
      http:
        http1MaxPendingRequests: 100
        maxRequestsPerConnection: 10
    outlierDetection:
      consecutive5xxErrors: 5
      interval: 30s
      baseEjectionTime: 30s
```

## 四、Serverless架构

### 4.1 AWS Lambda

```python
import json
import boto3

def lambda_handler(event, context):
    # 处理MQTT消息
    payload = json.loads(event['body'])
    
    # 写入DynamoDB
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table('sensor_data')
    
    table.put_item(Item={
        'device_id': payload['device_id'],
        'timestamp': payload['timestamp'],
        'temperature': payload['temperature'],
        'humidity': payload['humidity']
    })
    
    return {
        'statusCode': 200,
        'body': json.dumps({'message': 'Data stored successfully'})
    }
```

## 五、云安全

### 5.1 零信任架构

```yaml
# NetworkPolicy配置
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress

# 允许特定服务通信
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-sensor-service
spec:
  podSelector:
    matchLabels:
      app: sensor-service
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: api-gateway
    ports:
    - protocol: TCP
      port: 8080
```

## 六、基础设施即代码

### 6.1 Terraform配置

```hcl
# main.tf

provider "aws" {
  region = "us-east-1"
}

resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"
}

resource "aws_subnet" "public" {
  vpc_id     = aws_vpc.main.id
  cidr_block = "10.0.1.0/24"
  map_public_ip_on_launch = true
}

resource "aws_security_group" "sensor-sg" {
  name        = "sensor-sg"
  description = "Allow HTTP and MQTT"
  vpc_id      = aws_vpc.main.id

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    from_port   = 1883
    to_port     = 1883
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

resource "aws_eks_cluster" "sensor-cluster" {
  name     = "sensor-cluster"
  role_arn = aws_iam_role.eks-cluster-role.arn

  vpc_config {
    subnet_ids = [aws_subnet.public.id]
  }
}
```
