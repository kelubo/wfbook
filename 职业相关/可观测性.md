# 可观测性

## 概述

可观测性是指通过外部输出推断系统内部状态的能力，包括日志、指标和追踪三个维度。

## 可观测性三大支柱

```
┌─────────────────────────────────────────┐
│              日志（Logging）            │
│    记录系统事件和状态                   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│              指标（Metrics）            │
│    量化系统性能和状态                   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│              追踪（Tracing）            │
│    追踪请求在系统中的流转               │
└─────────────────────────────────────────┘
```

## 日志

### 日志收集

```bash
# 安装Filebeat
curl -L -O https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-7.17.0-amd64.deb
sudo dpkg -i filebeat-7.17.0-amd64.deb

# 配置Filebeat
filebeat.inputs:
- type: log
  enabled: true
  paths:
    - /var/log/*.log

output.elasticsearch:
  hosts: ["localhost:9200"]

# 启动Filebeat
sudo systemctl start filebeat
```

### 日志格式

```python
import logging
import json

logging.basicConfig(
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    level=logging.INFO
)

logger = logging.getLogger(__name__)

# 结构化日志
logger.info(json.dumps({
    'event': 'user_login',
    'user_id': 123,
    'status': 'success'
}))
```

### 日志分析

```bash
# 使用grep查找日志
grep "ERROR" /var/log/app.log

# 使用jq分析JSON日志
cat /var/log/app.log | jq '.user_id'

# 使用Elasticsearch查询
curl -X GET "localhost:9200/app-logs/_search?q=user_id:123"
```

## 指标

### 指标收集

```python
from prometheus_client import Counter, Gauge, Histogram, generate_latest
from flask import Flask

app = Flask(__name__)

# 计数器
REQUEST_COUNT = Counter('http_requests_total', 'Total HTTP requests')

# 仪表盘
REQUEST_LATENCY = Gauge('http_request_latency_seconds', 'HTTP request latency')

# 直方图
REQUEST_DURATION = Histogram('http_request_duration_seconds', 'HTTP request duration')

@app.before_request
def before_request():
    REQUEST_COUNT.inc()

@app.route('/metrics')
def metrics():
    return generate_latest()
```

### 指标类型

- **Counter**：单调递增计数器
- **Gauge**：可增可减仪表盘
- **Histogram**：分布直方图
- **Summary**：摘要统计

### 指标查询

```promql
# 查询请求总数
http_requests_total

# 查询请求速率
rate(http_requests_total[5m])

# 查询P99延迟
histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m]))

# 查询平均延迟
avg(http_request_latency_seconds)
```

## 追踪

### OpenTelemetry

```bash
# 安装OpenTelemetry
pip install opentelemetry-api opentelemetry-sdk opentelemetry-instrumentation-flask

# 配置追踪
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider

trace.set_tracer_provider(TracerProvider())
tracer = trace.get_tracer(__name__)

with tracer.start_as_current_span("hello"):
    print("Hello, World!")
```

### Jaeger

```bash
# 使用Docker安装Jaeger
docker run -d -p 16686:16686 -p 14268:14268 --name jaeger jaegertracing/all-in-one:latest

# 配置OpenTelemetry导出到Jaeger
from opentelemetry.exporter.jaeger.thrift import JaegerExporter
from opentelemetry.sdk.trace.export import BatchSpanProcessor

jaeger_exporter = JaegerExporter(
    agent_host_name="localhost",
    agent_port=6831,
)

trace.get_tracer_provider().add_span_processor(
    BatchSpanProcessor(jaeger_exporter)
)
```

### Zipkin

```bash
# 使用Docker安装Zipkin
docker run -d -p 9411:9411 --name zipkin openzipkin/zipkin

# 配置OpenTelemetry导出到Zipkin
from opentelemetry.exporter.zipkin.json import ZipkinExporter
from opentelemetry.sdk.trace.export import BatchSpanProcessor

zipkin_exporter = ZipkinExporter(
    endpoint="http://localhost:9411/api/v2/spans",
)

trace.get_tracer_provider().add_span_processor(
    BatchSpanProcessor(zipkin_exporter)
)
```

## 告警系统

### Alertmanager

```yaml
# alertmanager.yml
global:
  smtp_smarthost: 'smtp.example.com:587'
  smtp_from: 'alertmanager@example.com'

route:
  group_by: ['alertname']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  receiver: 'email'

receivers:
- name: 'email'
  email_configs:
  - to: 'admin@example.com'
```

### 告警规则

```yaml
# alert_rules.yml
groups:
- name: api_alerts
  rules:
  - alert: HighRequestLatency
    expr: avg(http_request_latency_seconds) > 0.5
    for: 5m
    labels:
      severity: critical
    annotations:
      summary: "High request latency on {{ $labels.instance }}"
      description: "Average latency is {{ $value }} seconds"
```

## 可观测性最佳实践

### 结构化日志

```python
# 使用JSON格式日志
import logging
from pythonjsonlogger import jsonlogger

logger = logging.getLogger()
handler = logging.StreamHandler()
formatter = jsonlogger.JsonFormatter('%(asctime)s %(levelname)s %(message)s')
handler.setFormatter(formatter)
logger.addHandler(handler)
```

### 统一标签

```python
# 使用统一标签
REQUEST_COUNT = Counter(
    'http_requests_total',
    'Total HTTP requests',
    ['endpoint', 'method', 'status_code']
)

REQUEST_COUNT.labels(endpoint='/api/users', method='GET', status_code=200).inc()
```

### 分布式追踪

```python
# 在微服务间传递追踪上下文
from opentelemetry import trace

def make_request(url):
    tracer = trace.get_tracer(__name__)
    with tracer.start_as_current_span("make_request"):
        response = requests.get(url)
        return response
```

### 仪表盘设计

```
1. 核心指标仪表盘：请求速率、延迟、错误率
2. 业务指标仪表盘：用户数、交易量、转化率
3. 基础设施仪表盘：CPU、内存、磁盘、网络
```

## 总结

可观测性是现代系统的核心能力，通过日志、指标和追踪三个维度，可以全面了解系统状态和性能。掌握可观测性技术对于故障排查和性能优化至关重要。
