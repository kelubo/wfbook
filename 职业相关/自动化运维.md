# 自动化运维

## 概述

自动化运维是通过工具和脚本自动执行运维任务，提高效率和可靠性。

## 自动化运维工具

### Ansible

#### 安装

```bash
# 安装Ansible
pip install ansible

# 验证安装
ansible --version
```

#### 基本配置

```yaml
# /etc/ansible/hosts
[web_servers]
server1 ansible_host=192.168.1.10
server2 ansible_host=192.168.1.11

[db_servers]
db1 ansible_host=192.168.1.20
```

#### 基本操作

```bash
# 测试连接
ansible all -m ping

# 执行命令
ansible web_servers -a "uptime"

# 安装软件
ansible web_servers -m apt -a "name=nginx state=present" --become

# 复制文件
ansible web_servers -m copy -a "src=/local/file dest=/remote/file"

# 执行剧本
ansible-playbook deploy.yml
```

#### Playbook

```yaml
# deploy.yml
---
- name: Deploy web application
  hosts: web_servers
  become: true
  
  tasks:
    - name: Install dependencies
      apt:
        name: "{{ item }}"
        state: present
      loop:
        - nginx
        - python3
        - python3-pip
    
    - name: Install Python packages
      pip:
        name: flask
    
    - name: Copy application files
      copy:
        src: app.py
        dest: /var/www/app.py
    
    - name: Start nginx
      service:
        name: nginx
        state: started
        enabled: true
```

### Terraform

#### 安装

```bash
# 安装Terraform
brew install terraform

# 验证安装
terraform version
```

#### 基本配置

```hcl
# main.tf
provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "web_server" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  
  tags = {
    Name = "WebServer"
  }
}
```

#### 基本操作

```bash
# 初始化
terraform init

# 查看计划
terraform plan

# 应用配置
terraform apply

# 销毁资源
terraform destroy

# 查看状态
terraform show
```

#### 变量和输出

```hcl
# variables.tf
variable "region" {
  description = "AWS region"
  type        = string
  default     = "us-east-1"
}

# outputs.tf
output "instance_ip" {
  value = aws_instance.web_server.public_ip
}
```

### Puppet

#### 安装

```bash
# 安装Puppet Server
sudo apt-get install puppetserver

# 安装Puppet Agent
sudo apt-get install puppet-agent
```

#### 基本配置

```puppet
# /etc/puppetlabs/code/environments/production/manifests/site.pp
node 'web_server' {
  include nginx
  include python
}

# /etc/puppetlabs/code/environments/production/modules/nginx/manifests/init.pp
class nginx {
  package { 'nginx':
    ensure => installed,
  }
  
  service { 'nginx':
    ensure => running,
    enable => true,
    require => Package['nginx'],
  }
}
```

#### 基本操作

```bash
# 运行Puppet Agent
puppet agent -t

# 查看资源状态
puppet resource package nginx

# 查看配置
puppet config print
```

## 自动化运维实践

### 服务器配置

```yaml
# ansible-playbook configure.yml
---
- name: Configure servers
  hosts: all
  become: true
  
  tasks:
    - name: Update system
      apt:
        update_cache: yes
        upgrade: dist
    
    - name: Install common packages
      apt:
        name: "{{ item }}"
        state: present
      loop:
        - vim
        - htop
        - tmux
    
    - name: Configure SSH
      template:
        src: sshd_config.j2
        dest: /etc/ssh/sshd_config
      notify: Restart SSH
    
    handlers:
      - name: Restart SSH
        service:
          name: sshd
          state: restarted
```

### 应用部署

```yaml
# ansible-playbook deploy.yml
---
- name: Deploy application
  hosts: web_servers
  become: true
  
  tasks:
    - name: Pull latest code
      git:
        repo: 'https://github.com/user/app.git'
        dest: /var/www/app
        version: main
    
    - name: Install dependencies
      pip:
        requirements: /var/www/app/requirements.txt
    
    - name: Restart application
      service:
        name: myapp
        state: restarted
```

### 数据库管理

```yaml
# ansible-playbook database.yml
---
- name: Manage database
  hosts: db_servers
  become: true
  
  tasks:
    - name: Create database
      postgresql_db:
        name: mydb
        state: present
    
    - name: Create user
      postgresql_user:
        name: myuser
        password: mypassword
        db: mydb
        priv: ALL
    
    - name: Backup database
      command: pg_dump mydb > /backup/mydb_backup.sql
```

```yaml
- name: Configure monitoring

        name: prometheus-node-exporter
        state: started
        enabled: true
```

### 告警自动化

```python
# alert_auto.py
import smtplib
from email.mime.text import MIMEText

def send_alert(subject, message):
    msg = MIMEText(message)
    msg['Subject'] = subject
    msg['From'] = 'alert@example.com'
    msg['To'] = 'admin@example.com'
    
    with smtplib.SMTP('smtp.example.com') as server:
        server.send_message(msg)

# 检查CPU使用率
cpu_usage = get_cpu_usage()
if cpu_usage > 90:
    send_alert('High CPU Usage', f'CPU usage is {cpu_usage}%')
```

## 总结

自动化运维是现代运维的核心实践，通过Ansible、Terraform等工具，可以实现服务器配置、应用部署和基础设施管理的自动化。掌握自动化运维技术可以显著提高运维效率和可靠性。
