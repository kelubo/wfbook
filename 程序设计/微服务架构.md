# 微服务架构

## 概述

微服务架构是一种将应用程序拆分为多个独立服务的架构风格，每个服务运行在自己的进程中，通过轻量级通信机制（如RESTful API、gRPC）进行交互。

## 微服务架构特点

- **单一职责**：每个服务只负责一个业务功能
- **独立部署**：每个服务可以独立部署和升级
- **独立团队**：每个服务由独立的团队负责
- **技术多样性**：不同服务可以使用不同的技术栈
- **弹性扩展**：可以根据需求独立扩展每个服务

## 微服务架构设计

### 服务拆分原则

1. **按业务边界拆分**：根据业务领域划分服务
2. **避免共享数据库**：每个服务拥有独立的数据库
3. **高内聚低耦合**：服务内部高内聚，服务间低耦合
4. **接口稳定**：定义稳定的API接口

#### gRPC

```protobuf
// user.proto
syntax = "proto3";

service UserService {
    rpc GetUser(GetUserRequest) returns (GetUserResponse);
    rpc CreateUser(CreateUserRequest) returns (CreateUserResponse);
}

message GetUserRequest {
    int32 user_id = 1;
}

message GetUserResponse {
    int32 id = 1;
    string name = 2;
    string email = 3;
}

message CreateUserRequest {
    string name = 1;
    string email = 2;
}

message CreateUserResponse {
    int32 id = 1;
}
```

```python
# gRPC服务端
import grpc
from concurrent import futures
import user_pb2
import user_pb2_grpc

class UserService(user_pb2_grpc.UserServiceServicer):
    def GetUser(self, request, context):
        return user_pb2.GetUserResponse(
            id=request.user_id,
            name='John Doe',
            email='john@example.com'
        )

server = grpc.server(futures.ThreadPoolExecutor(max_workers=10))
user_pb2_grpc.add_UserServiceServicer_to_server(UserService(), server)
server.add_insecure_port('[::]:50051')
server.start()
```

#### 消息队列

```python
import pika

# 生产者
connection = pika.BlockingConnection(pika.ConnectionParameters('localhost'))
channel = connection.channel()
channel.queue_declare(queue='task_queue', durable=True)

channel.basic_publish(
    exchange='',
    routing_key='task_queue',
    body='Hello World!',
    properties=pika.BasicProperties(delivery_mode=2)
)
connection.close()

# 消费者
def callback(ch, method, properties, body):
    print(f"Received {body}")
    ch.basic_ack(delivery_tag=method.delivery_tag)

channel.basic_consume(queue='task_queue', on_message_callback=callback)
channel.start_consuming()
```

## 微服务安全

### 认证与授权

```python
# JWT认证示例
import jwt
from datetime import datetime, timedelta

def generate_token(user_id):
    payload = {
        'user_id': user_id,
        'exp': datetime.utcnow() + timedelta(hours=24)
    }
    token = jwt.encode(payload, 'secret_key', algorithm='HS256')
    return token

def verify_token(token):
    try:
        payload = jwt.decode(token, 'secret_key', algorithms=['HS256'])
        return payload['user_id']
    except jwt.ExpiredSignatureError:
        return None
    except jwt.InvalidTokenError:
        return None
```

## 微服务挑战

### 分布式事务

- **两阶段提交（2PC）**：协调者和参与者
- **补偿事务（Saga）**：一系列本地事务
- **最终一致性**：通过消息队列实现

### 服务间调用链

- **分布式追踪**：Zipkin、Jaeger
- **链路可视化**：Grafana Tempo

### 数据一致性

- **事件驱动架构**：通过事件同步数据
- **CQRS**：命令查询职责分离
- **Event Sourcing**：事件溯源

## 总结

微服务架构通过将应用拆分为独立服务，提高了系统的可扩展性、可维护性和部署灵活性。但也带来了分布式系统的复杂性，需要解决服务发现、通信、监控、安全等问题。
