# 前端工程化指南

## 概述

前端工程化是将软件工程的方法和工具应用于前端开发，提高开发效率和代码质量。

---

## 一、构建工具

### 1.1 Webpack

```
Webpack核心概念
        │
        ├─ Entry：入口文件
        ├─ Output：输出配置
        ├─ Loader：转换非JavaScript文件
        ├─ Plugin：扩展Webpack功能
        └─ Mode：模式（development/production）
```

```
Webpack配置示例
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.[contenthash].js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      },
      {
        test: /\.(js|jsx)$/,
        exclude: /node_modules/,
        use: 'babel-loader'
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ],
  mode: 'production'
}
```

### 1.2 Vite

```
Vite核心概念
        │
        ├─ 快速冷启动：利用ES Module
        ├─ 按需编译：只编译需要的模块
        ├─ 热模块替换：HMR即时更新
        └─ 原生支持：TypeScript、JSX、CSS
```

```
Vite配置示例
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['vue', 'vue-router']
        }
      }
    }
  }
})
```

### 1.3 Rollup

```
Rollup特点
        │
        ├─ 专注于打包JavaScript库
        ├─ Tree Shaking：移除未使用代码
        ├─ 输出格式：ESM、CommonJS、UMD
        └─ 插件丰富：支持各种转换
```

## 二、代码质量工具

### 2.1 ESLint

```
ESLint配置
        │
        ├─ 规则：代码规范规则
        ├─ 插件：扩展规则
        ├─ 解析器：解析JavaScript/TypeScript
        └─ 配置文件：.eslintrc.js
```

```
ESLint配置示例
module.exports = {
  env: {
    browser: true,
    es2021: true
  },
  extends: [
    'eslint:recommended',
    'plugin:vue/essential'
  ],
  parserOptions: {
    ecmaVersion: 'latest',
    sourceType: 'module'
  },
  plugins: ['vue'],
  rules: {
    'no-console': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    'no-debugger': process.env.NODE_ENV === 'production' ? 'warn' : 'off'
  }
}
```

### 2.2 Prettier

```
Prettier配置
        │
        ├─ printWidth：每行最大字符数
        ├─ tabWidth：缩进宽度
        ├─ singleQuote：使用单引号
        ├─ trailingComma：尾随逗号
        └─ semi：是否添加分号
```

```
Prettier配置示例
{
  "printWidth": 80,
  "tabWidth": 2,
  "singleQuote": true,
  "trailingComma": "es5",
  "semi": true
}
```

## 四、测试框架

### 4.1 Jest

```
Jest特点
        │
        ├─ 零配置：开箱即用
        ├─ 快照测试：UI测试
        ├─ Mock功能：模拟依赖
        └─ 覆盖率报告：代码覆盖率
```

```
Jest测试示例
test('adds 1 + 2 to equal 3', () => {
  expect(1 + 2).toBe(3)
})

test('user object has correct properties', () => {
  const user = { name: 'John', age: 30 }
  expect(user).toEqual({ name: 'John', age: 30 })
})
```

### 4.2 Vue Test Utils

```
Vue组件测试示例
import { mount } from '@vue/test-utils'
import HelloWorld from './HelloWorld.vue'

test('renders props.msg when passed', () => {
  const msg = 'new message'
  const wrapper = mount(HelloWorld, {
    propsData: { msg }
  })
  expect(wrapper.text()).toMatch(msg)
})
```

## 五、自动化部署

### 5.1 CI/CD流程

```
CI/CD流程
        │
        ├─ 代码提交：git push
        ├─ 自动化构建：npm run build
        ├─ 自动化测试：npm test
        ├─ 代码质量检查：ESLint、Prettier
        ├─ 部署到测试环境
        └─ 部署到生产环境
```

### 5.2 GitHub Actions

```
GitHub Actions配置示例
name: CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Use Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 18
      - run: npm ci
      - run: npm run build
      - run: npm test
```

### 5.3 静态站点部署

静态站点部署平台
