# 时序数据库

## 概述

时序数据库是专门用于存储和查询时间序列数据的数据库系统，时间序列数据是按时间顺序记录的数据点。

## 时序数据特点

- **时间有序**：数据按时间戳排序
- **高写入量**：每秒百万级数据点
- **高查询量**：实时查询和聚合
- **数据压缩**：存储空间优化
- **时间窗口**：按时间范围查询

## 时序数据库对比

| 特性 | InfluxDB | TimescaleDB | Prometheus |
|------|----------|-------------|------------|
| 数据模型 | 时间序列、标签 | 关系型表 | 指标、标签 |
| 查询语言 | Flux/InfluxQL | SQL | PromQL |
| 存储引擎 | 自定义 | PostgreSQL | TSDB |
| 分布式 | 支持 | 支持 | 有限 |
| 数据保留 | 自动策略 | 支持 | 配置保留 |

## InfluxDB

### 安装

```bash
# 使用Docker安装
docker run -d -p 8086:8086 --name influxdb influxdb:2.0

# 登录
# 访问 http://localhost:8086
```

### 基本操作

```bash
# 创建Bucket
influx bucket create -n sensor_data -o myorg

# 写入数据
influx write \
  -b sensor_data \
  -o myorg \
  'temperature,device=sensor1 value=25.5 1609459200000000000'

# 查询数据
influx query \
  -o myorg \
  'from(bucket: "sensor_data") |> range(start: -1h)'
```

### Flux查询语言

```flux
// 查询最近1小时的温度数据
from(bucket: "sensor_data")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "temperature")
  |> filter(fn: (r) => r.device == "sensor1")
  |> mean()

// 计算每5分钟的平均值
from(bucket: "sensor_data")
  |> range(start: -24h)
  |> filter(fn: (r) => r._measurement == "temperature")
  |> aggregateWindow(every: 5m, fn: mean)

// 异常检测
from(bucket: "sensor_data")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "temperature")
  |> anomalyDetection(method: "grubbs")
```

### Python客户端

```python
from influxdb_client import InfluxDBClient, Point

client = InfluxDBClient(url="http://localhost:8086", token="my-token", org="myorg")

# 写入数据
write_api = client.write_api()
point = Point("temperature").tag("device", "sensor1").field("value", 25.5)
write_api.write(bucket="sensor_data", record=point)

# 查询数据
query_api = client.query_api()
tables = query_api.query('from(bucket: "sensor_data") |> range(start: -1h)')

for table in tables:
    for record in table.records:
        print(record.get_value())
```

## TimescaleDB

### 安装

```bash
# 使用Docker安装
docker run -d -p 5432:5432 --name timescaledb timescale/timescaledb:latest-pg13

# 创建数据库
createdb sensor_db
```

### 基本操作

```sql
-- 创建超表
CREATE TABLE sensor_data (
    time TIMESTAMPTZ NOT NULL,
    device TEXT NOT NULL,
    temperature DOUBLE PRECISION,
    humidity DOUBLE PRECISION
);

SELECT create_hypertable('sensor_data', 'time');

-- 插入数据
INSERT INTO sensor_data (time, device, temperature, humidity)
VALUES (NOW(), 'sensor1', 25.5, 60);

-- 查询数据
SELECT * FROM sensor_data WHERE time >= NOW() - INTERVAL '1 hour';

-- 按时间聚合
SELECT time_bucket('5 minutes', time) AS bucket,
       AVG(temperature) AS avg_temp
FROM sensor_data
WHERE time >= NOW() - INTERVAL '24 hours'
GROUP BY bucket
ORDER BY bucket;
```

### Python客户端

```python
import psycopg2

conn = psycopg2.connect(
    host='localhost',
    user='postgres',
    password='password',
    database='sensor_db'
)

cursor = conn.cursor()

# 插入数据
cursor.execute('''
    INSERT INTO sensor_data (time, device, temperature, humidity)
    VALUES (NOW(), %s, %s, %s)
''', ('sensor1', 25.5, 60))
conn.commit()

# 查询数据
cursor.execute('''
    SELECT * FROM sensor_data WHERE time >= NOW() - INTERVAL '1 hour'
''')
results = cursor.fetchall()

for row in results:
    print(row)

conn.close()
```

## Prometheus

### 安装

```bash
# 使用Docker安装
docker run -d -p 9090:9090 --name prometheus prom/prometheus

# 配置文件
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'node_exporter'
    static_configs:
      - targets: ['localhost:9100']
```

### PromQL查询

```promql
# 查询最近5分钟的CPU使用率
100 - (avg by (instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# 查询内存使用率
100 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes * 100)

# 查询网络带宽
rate(node_network_receive_bytes_total[1m])

# 计算每5分钟的平均值
avg_over_time(node_cpu_seconds_total[5m])

# 聚合多个指标
sum(rate(node_network_receive_bytes_total[1m])) by (instance)
```

### Remote Write


### 自定义采集器

```python
import time
from influxdb_client import InfluxDBClient, Point

client = InfluxDBClient(url="http://localhost:8086", token="my-token", org="myorg")
write_api = client.write_api()

while True:
    # 模拟传感器数据
    temperature = 25.0 + (time.time() % 10) / 10
    humidity = 60.0 + (time.time() % 5) / 5
    
    point = Point("sensor") \
        .tag("device", "sensor1") \
        .field("temperature", temperature) \
        .field("humidity", humidity)
    
    write_api.write(bucket="sensor_data", record=point)
    time.sleep(1)
```

```
温度、湿度、压力等传感器实时数据
```


### 金融数据分析

```
股票价格、交易数据的时间序列分析
```

### IoT数据存储

```
物联网设备的实时数据流
```

## 时序数据库选择建议

### 选择InfluxDB的情况

- 需要高性能写入和查询
- 需要强大的数据压缩
- 需要灵活的数据保留策略

### 选择TimescaleDB的情况

- 已经在使用PostgreSQL
- 需要SQL查询能力
- 需要复杂的数据分析

### 选择Prometheus的情况

- 需要监控系统指标
- 需要与Grafana集成
- 需要灵活的查询语言

## 总结

时序数据库是存储时间序列数据的最佳选择，InfluxDB、TimescaleDB和Prometheus各有其特点和适用场景。选择合适的时序数据库取决于数据规模、查询需求和技术栈。
