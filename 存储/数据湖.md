# 数据湖

## 概述

数据湖是一种存储架构，用于存储结构化、半结构化和非结构化数据的原始格式，支持多种分析和处理引擎。

## 数据湖架构

```
┌─────────────────────────────────────────┐
│              数据采集层                 │
│    IoT设备、日志、数据库、API          │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│              数据存储层                 │
│    对象存储（S3、MinIO、HDFS）         │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│              数据处理层                 │
│    Spark、Flink、Presto、Hive         │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│              数据分析层                 │
│    BI工具、机器学习、数据可视化        │
└─────────────────────────────────────────┘
```

## 数据湖特点

- **存储原始数据**：保留数据原始格式
- **支持多种数据类型**：结构化、半结构化、非结构化
- **弹性扩展**：按需扩展存储容量
- **成本效益**：低成本存储方案
- **支持多种分析引擎**：Spark、Flink、Presto等

## 对象存储

### MinIO

#### 安装

```bash
# 使用Docker安装
docker run -d -p 9000:9000 --name minio \
    -e "MINIO_ROOT_USER=admin" \
    -e "MINIO_ROOT_PASSWORD=password" \
    -v /data/minio:/data \
    minio/minio server /data --console-address ":9001"

# 登录
# 访问 http://localhost:9001
```

#### 基本操作

```bash
# 创建Bucket
mc mb myminio/sensor-data

# 上传文件
mc cp data.csv myminio/sensor-data/

# 下载文件
mc cp myminio/sensor-data/data.csv .

# 列出文件
mc ls myminio/sensor-data/

# 删除文件
mc rm myminio/sensor-data/data.csv
```

#### Python客户端

```python
from minio import Minio

client = Minio(
    "localhost:9000",
    access_key="admin",
    secret_key="password",
    secure=False
)

# 创建Bucket
client.make_bucket("sensor-data")

# 上传文件
client.fput_object("sensor-data", "data.csv", "/path/to/data.csv")

# 下载文件
client.fget_object("sensor-data", "data.csv", "/path/to/download.csv")

# 列出文件
objects = client.list_objects("sensor-data")
for obj in objects:
    print(obj.object_name)
```

## 数据处理引擎

### Apache Spark

```bash
# 安装Spark
brew install apache-spark

# 启动Spark Shell
spark-shell

# Spark SQL
spark.sql("SELECT * FROM parquet.`s3://sensor-data/data.parquet`").show()
```

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("DataLake") \
    .getOrCreate()

# 读取CSV文件
df = spark.read.csv("s3://sensor-data/data.csv", header=True, inferSchema=True)

# 数据处理
df_filtered = df.filter(df["temperature"] > 25)

# 写入Parquet
df_filtered.write.parquet("s3://sensor-data/data_filtered.parquet")

# Spark SQL
df.createOrReplaceTempView("sensor_data")
result = spark.sql("SELECT device, AVG(temperature) FROM sensor_data GROUP BY device")
result.show()
```

### Apache Flink

```bash
# 安装Flink
brew install apache-flink

# 启动Flink
./bin/start-cluster.sh

# 提交作业
./bin/flink run ./examples/streaming/WordCount.jar
```

```python
from pyflink.datastream import StreamExecutionEnvironment

env = StreamExecutionEnvironment.get_execution_environment()

# 读取数据流
ds = env.read_text_file("s3://sensor-data/stream/*.log")

# 数据处理
result = ds.flat_map(lambda x: x.split()) \
           .map(lambda x: (x, 1)) \
           .key_by(lambda x: x[0]) \
           .sum(1)

# 输出结果
result.print()

env.execute("DataLake")
```

## 数据湖格式

### Parquet

```python
# 写入Parquet
df.write.parquet("s3://sensor-data/data.parquet", mode="overwrite")

# 读取Parquet
df = spark.read.parquet("s3://sensor-data/data.parquet")
```

### ORC

```python
# 写入ORC
df.write.orc("s3://sensor-data/data.orc", mode="overwrite")

# 读取ORC
df = spark.read.orc("s3://sensor-data/data.orc")
```

### Avro

```python
# 写入Avro
df.write.format("avro").save("s3://sensor-data/data.avro")

# 读取Avro
df = spark.read.format("avro").load("s3://sensor-data/data.avro")
```

## 数据湖安全

### 访问控制

```bash
# MinIO策略
mc admin policy add myminio read-only ./read-only.json

# JSON策略文件
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": ["s3:GetObject"],
            "Resource": ["arn:aws:s3:::sensor-data/*"]
        }
    ]
}
```

### 数据加密

```bash
# MinIO服务端加密
mc admin config set myminio encrypt/state "on"

# 客户端加密
mc cp --encrypt data.csv myminio/sensor-data/
```

## 数据湖最佳实践

### 数据分区

```python
# 按日期分区
df.write.partitionBy("date").parquet("s3://sensor-data/data.parquet")

# 查询特定分区
df = spark.read.parquet("s3://sensor-data/data.parquet/date=2024-01-01")
```

### 数据压缩

```python
# Snappy压缩
df.write.option("compression", "snappy").parquet("s3://sensor-data/data.parquet")

# Gzip压缩
df.write.option("compression", "gzip").parquet("s3://sensor-data/data.parquet")
```

### 数据质量

```python
# 数据质量检查
from great_expectations.dataset import SparkDFDataset

ge_df = SparkDFDataset(df)
result = ge_df.expect_column_values_to_be_between("temperature", min_value=-40, max_value=100)
print(result)
```

## 总结

数据湖是存储和分析大规模数据的有效架构，通过对象存储和多种处理引擎，可以实现灵活的数据存储和分析。选择合适的技术栈取决于数据规模、处理需求和成本预算。
