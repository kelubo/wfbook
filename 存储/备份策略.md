# 备份策略

## 概述

备份策略是确保数据安全和可恢复性的关键措施，包括备份频率、备份类型、存储位置和恢复流程。

## 备份原则

### 3-2-1原则

```
3份数据副本
2种不同存储介质
1份异地备份
```

### 备份类型

| 类型 | 描述 | 频率 |
|------|------|------|
| 全量备份 | 备份所有数据 | 每周/每月 |
| 增量备份 | 备份上次备份后变化的数据 | 每天 |
| 差异备份 | 备份上次全量备份后变化的数据 | 每天 |

### RPO和RTO

- **RPO（恢复点目标）**：可接受的数据丢失量
- **RTO（恢复时间目标）**：恢复所需的时间

## 备份策略设计

### 备份频率

```
全量备份：每周日凌晨2点
增量备份：周一至周六凌晨2点
差异备份：可选，视数据变化率而定
```

### 备份保留

```
全量备份：保留4周
增量备份：保留1周
差异备份：保留1周
```

### 备份存储

```
本地存储：SSD/HDD
异地存储：云存储（S3、MinIO）
磁带存储：长期归档
```

## 数据库备份

### MySQL备份

```bash
# 全量备份
mysqldump -u root -p --all-databases > full_backup.sql

# 增量备份（需要开启binlog）
mysqlbinlog binlog.000001 > incremental_backup.sql

# 使用xtrabackup
xtrabackup --backup --target-dir=/backup/full

# 恢复全量备份
mysql -u root -p < full_backup.sql

# 恢复增量备份
mysqlbinlog binlog.000001 | mysql -u root -p
```

### PostgreSQL备份

```bash
# 全量备份
pg_dump -U postgres --all-databases > full_backup.sql

# 使用pg_basebackup
pg_basebackup -D /backup/full -X stream

# 恢复全量备份
psql -U postgres -f full_backup.sql

# 时间点恢复
pg_waldump pg_wal/000000010000000000000001 > wal_backup.sql
```

### MongoDB备份

```bash
# 全量备份
mongodump --out /backup/full

# 增量备份（需要开启oplog）
mongodump --oplog --out /backup/incremental

# 恢复全量备份
mongorestore /backup/full

# 恢复增量备份
mongorestore --oplogReplay /backup/incremental
```

## 文件备份

### rsync备份

```bash
# 全量备份
rsync -av /data /backup/full

# 增量备份
rsync -av --link-dest=/backup/full /data /backup/incremental

# 定时任务
0 2 * * * rsync -av /data /backup/daily/$(date +%Y%m%d)
```

### tar备份

```bash
# 全量备份
tar -czf /backup/full_$(date +%Y%m%d).tar.gz /data

# 增量备份
tar -czf /backup/incremental_$(date +%Y%m%d).tar.gz \
    --newer-mtime="$(date -d '1 day ago' +%Y-%m-%d)" /data
```

### BorgBackup

```bash
# 安装BorgBackup
pip install borgbackup

# 创建仓库
borg init --encryption=repokey /backup/borg

# 全量备份
borg create /backup/borg::full_$(date +%Y%m%d) /data

# 增量备份
borg create /backup/borg::incremental_$(date +%Y%m%d) /data \
    --parent /backup/borg::full_$(date -d '1 week ago' +%Y%m%d)

# 查看备份列表
borg list /backup/borg

# 恢复备份
borg extract /backup/borg::full_20240101
```

## 云备份

### AWS S3备份

```bash
# 安装AWS CLI
pip install awscli

# 配置AWS CLI
aws configure

# 同步到S3
aws s3 sync /data s3://my-backup-bucket/data

# 定时同步
0 2 * * * aws s3 sync /data s3://my-backup-bucket/data

# 生命周期策略
{
    "Rules": [
        {
            "ID": "ExpireOldBackups",
            "Prefix": "data/",
            "Status": "Enabled",
            "Expiration": {
                "Days": 30
            }
        }
    ]
}
```

### Azure Blob备份

```bash

curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

# 登录Azure
az login

# 创建存储账户

# 定时同步
0 2 * * * az storage blob upload-batch --source /data --destination mycontainer
```

## 备份验证

### 数据库备份验证

```bash
# MySQL验证
mysql -u root -p < full_backup.sql 2>&1 | grep -i error

# PostgreSQL验证
psql -U postgres -f full_backup.sql 2>&1 | grep -i error

# MongoDB验证
mongorestore --dryRun /backup/full
```

### 文件备份验证

```bash
# 校验tar文件
tar -tzf backup.tar.gz | head -10

# 校验rsync
rsync -av --dry-run /data /backup/full

# 校验BorgBackup
borg check /backup/borg
```

## 灾难恢复

```

4. 执行恢复操作
5. 验证恢复结果
6. 测试系统功能
7. 恢复用户访问
```

### 告警配置

```bash
# 备份失败告警
if [ $? -ne 0 ]; then
    echo "Backup failed" | mail -s "Backup Alert" admin@example.com
fi

# 备份大小异常告警
backup_size=$(du -sh /backup | awk '{print $1}')
if [ "$backup_size" > "100G" ]; then
    echo "Backup size exceeded" | mail -s "Storage Alert" admin@example.com
fi
```
