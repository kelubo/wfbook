# 大数据处理框架

## 概述

大数据处理框架是处理海量数据的工具和平台，包括批处理、流处理和交互式查询等模式。

---

## 一、Hadoop生态系统

### 1.1 Hadoop核心组件

```
Hadoop核心组件
        │
        ├─ HDFS（分布式文件系统）
        │   ├─ NameNode：管理文件系统元数据
        │   ├─ DataNode：存储实际数据块
        │   ├─ SecondaryNameNode：辅助NameNode
        │   └─ 数据块大小：默认128MB
        │
        ├─ YARN（资源管理器）
        │   ├─ ResourceManager：全局资源管理
        │   ├─ NodeManager：节点资源管理
        │   └─ ApplicationMaster：应用程序管理
        │
        └─ MapReduce（批处理框架）
            ├─ Map阶段：数据分片处理
            ├─ Shuffle阶段：数据重排
            └─ Reduce阶段：汇总结果
```

### 1.2 Hadoop生态系统组件

```
Hadoop生态系统组件
        │
        ├─ HBase：分布式列式数据库
        ├─ Hive：数据仓库工具，类SQL查询
        ├─ Pig：数据流语言
        ├─ Zookeeper：分布式协调服务
        ├─ Flume：日志收集工具
        ├─ Sqoop：数据迁移工具
        ├─ Oozie：工作流调度
        └─ Ambari：集群管理
```

## 二、Spark

### 2.1 Spark核心概念

```
Spark核心概念
        │
        ├─ RDD（弹性分布式数据集）
        │   ├─ 不可变的分布式数据集合
        │   ├─ 支持多种分区策略
        │   ├─ 惰性求值
        │   └─ 容错机制：Lineage
        │
        ├─ DataFrame：结构化数据集合
        ├─ Dataset：强类型数据集合
        ├─ Spark SQL：SQL查询接口
        └─ Spark Streaming：流处理
```

### 2.2 Spark组件

```
Spark组件
        │
        ├─ Spark Core：核心引擎
        ├─ Spark SQL：结构化数据处理
        ├─ Spark Streaming：流处理
        ├─ MLlib：机器学习库
        ├─ GraphX：图计算
        └─ Structured Streaming：结构化流处理
```

### 2.3 Spark编程模型

```
Spark编程模型示例（Scala）
import org.apache.spark.SparkContext

val sc = new SparkContext("local", "WordCount")
val textFile = sc.textFile("hdfs://path/to/file")
val wordCounts = textFile.flatMap(line => line.split(" "))
                        .map(word => (word, 1))
                        .reduceByKey(_ + _)
wordCounts.saveAsTextFile("hdfs://path/to/output")
sc.stop()
```

## 三、Flink

### 3.1 Flink核心概念

```
Flink核心概念
        │
        ├─ 数据流模型：有界流和无界流
        ├─ 状态管理：键控状态和算子状态
        ├─ 窗口机制：时间窗口和计数窗口
        ├─ 容错机制：Checkpoint和Savepoint
        └─ 事件时间：基于事件产生的时间
```

### 3.2 Flink架构

```
Flink架构
        │
        ├─ JobManager：管理作业执行
        ├─ TaskManager：执行任务
        ├─ ResourceManager：资源管理
        └─ Dispatcher：接收作业提交
```

### 3.3 Flink编程模型

```
Flink编程模型示例（Java）
StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();

DataStream<String> text = env.socketTextStream("localhost", 9999);

DataStream<Tuple2<String, Integer>> wordCounts = text
    .flatMap(new Tokenizer())
    .keyBy(0)
    .timeWindow(Time.seconds(5))
    .sum(1);

wordCounts.print();

env.execute("WordCount");
```

## 四、流处理框架对比

```
流处理框架对比
        │
        ├─ Spark Streaming
        │   ├─ 微批处理：将流数据分成小批次处理
        │   ├─ 延迟：秒级
        │   └─ 适用场景：对延迟要求不高的场景
        │
        ├─ Flink
        │   ├─ 真正的流处理：逐条处理数据
        │   ├─ 延迟：毫秒级
        │   └─ 适用场景：对延迟要求高的场景
        │
        ├─ Storm
        │   ├─ 真正的流处理：逐条处理数据
        │   ├─ 延迟：毫秒级
        │   └─ 适用场景：实时计算
        │
        └─ Kafka Streams
            ├─ 基于Kafka的流处理
            ├─ 轻量级：无需单独集群
            └─ 适用场景：与Kafka集成的场景
```

## 七、大数据架构

### 7.1 Lambda架构

```
Lambda架构
        │
        ├─ 批处理层：处理历史数据
        │   ├─ 数据源：原始数据
        │   ├─ 批处理：MapReduce/Spark
        │   └─ 批视图：预计算结果
        │
        ├─ 速度层：处理实时数据
        │   ├─ 数据源：实时数据
        │   ├─ 流处理：Storm/Flink
        │   └─ 实时视图：实时计算结果
        │
        └─ 服务层：统一查询接口
            └─ 合并批视图和实时视图
```

### 7.2 Kappa架构

```
Kappa架构
        │
        ├─ 用流处理统一批处理和流处理
        │   ├─ 数据源：统一的日志
        │   ├─ 流处理：Flink/Spark Streaming
        │   └─ 视图：实时更新
        ├─ 优点：简化架构，统一处理逻辑
        └─ 缺点：对实时处理要求高
```

## 八、大数据处理流程

```
大数据处理流程
        │
        ├─ 数据采集：收集原始数据
        ├─ 数据存储：存储到分布式文件系统
        ├─ 数据处理：批处理或流处理
        ├─ 数据分析：挖掘数据价值
        ├─ 数据可视化：展示分析结果
        └─ 数据应用：应用到业务场景
```

## 总结

大数据处理框架是处理海量数据的核心工具，通过Hadoop、Spark、Flink等框架，可以实现高效的数据处理和分析。选择合适的框架需要根据业务需求和数据特点来决定。
