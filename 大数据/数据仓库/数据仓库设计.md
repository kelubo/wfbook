# 数据仓库设计

## 概述

数据仓库是一个面向主题的、集成的、非易失的、随时间变化的数据集合，用于支持管理决策。

---

## 一、数据仓库概念

### 1.1 数据仓库特点

```
数据仓库特点
        │
        ├─ 面向主题：围绕业务主题组织数据
        ├─ 集成：来自多个数据源的统一数据
        ├─ 非易失：数据一旦写入不修改
        ├─ 随时间变化：数据带有时间戳
        └─ 面向分析：支持复杂查询和分析
```

### 1.2 数据仓库与数据库的区别

```
数据仓库与数据库区别
        │
        ├─ 数据库
        │   ├─ 面向操作：OLTP
        │   ├─ 数据更新：频繁增删改
        │   ├─ 数据粒度：细粒度
        │   ├─ 查询类型：简单查询
        │   └─ 数据量：相对较小
        │
        └─ 数据仓库
            ├─ 面向分析：OLAP
            ├─ 数据更新：批量加载
            ├─ 数据粒度：多粒度
            ├─ 查询类型：复杂查询
            └─ 数据量：海量数据
```

## 二、数据仓库架构

### 2.1 三层架构

```
数据仓库三层架构
        │
        ├─ 数据源层：原始数据来源
        │   ├─ 业务数据库
        │   ├─ 日志文件
        │   ├─ 外部数据
        │   └─ 实时数据
        │
        ├─ 数据仓库层：数据存储和处理
        │   ├─ ODS：操作数据存储
        │   ├─ DWD：明细数据层
        │   ├─ DWS：汇总数据层
        │   └─ ADS：应用数据层
        │
        └─ 数据服务层：数据应用和展示
            ├─ BI工具
            ├─ 数据API
            ├─ 数据挖掘
            └─ 报表系统
```

### 2.2 数据分层

```
数据分层设计
        │
        ├─ ODS（操作数据存储）
        │   ├─ 原始数据原样存储
        │   ├─ 保留数据原貌
        │   └─ 用于数据追溯
        │
        ├─ DWD（明细数据层）
        │   ├─ 清洗和转换后的数据
        │   ├─ 去除冗余和噪声
        │   └─ 保留业务明细
        │
        ├─ DWS（汇总数据层）
        │   ├─ 按主题汇总的数据
        │   ├─ 支持多维分析
        │   └─ 提高查询效率
        │
        └─ ADS（应用数据层）
            ├─ 面向具体应用的数据
            ├─ 定制化的数据
            └─ 直接用于报表和分析
```

## 三、数据建模

### 3.1 星型模型

```
星型模型结构
        │
        ├─ 事实表：业务度量数据
        │   ├─ 主键：多个维度主键组合
        │   ├─ 度量值：数值型数据
        │   └─ 时间戳：业务时间
        │
        └─ 维度表：描述性数据
            ├─ 维度主键
            ├─ 维度属性
            └─ 层级结构
```

```
星型模型示例（销售分析）
        │
        ├─ 事实表：销售事实表
        │   ├─ 订单ID
        │   ├─ 产品ID
        │   ├─ 客户ID
        │   ├─ 时间ID
        │   ├─ 销售额
        │   ├─ 销售数量
        │   └─ 利润
        │
        ├─ 维度表：产品维度表
        │   ├─ 产品ID
        │   ├─ 产品名称
        │   ├─ 产品类别
        │   └─ 产品价格
        │
        ├─ 维度表：客户维度表
        │   ├─ 客户ID
        │   ├─ 客户名称
        │   ├─ 客户地区
        │   └─ 客户等级
        │
        └─ 维度表：时间维度表
            ├─ 时间ID
            ├─ 日期
            ├─ 月份
            ├─ 季度
            └─ 年份
```

### 3.2 雪花模型

```
雪花模型结构
        │
        ├─ 事实表：业务度量数据
        │
        └─ 维度表：规范化的维度表
            ├─ 一级维度表
            ├─ 二级维度表（规范化）
            └─ 三级维度表（规范化）
```

### 3.3 星座模型

```
星座模型结构
        │
        ├─ 多个事实表共享维度表
        │
        └─ 共享维度表：多个事实表共用的维度
```

### 3.4 模型对比

```
数据模型对比
        │
        ├─ 星型模型
        │   ├─ 优点：查询简单，性能好
        │   └─ 缺点：数据冗余
        │
        ├─ 雪花模型
        │   ├─ 优点：数据规范化，减少冗余
        │   └─ 缺点：查询复杂，性能差
        │
        └─ 星座模型
            ├─ 优点：共享维度，一致性好
            └─ 缺点：复杂度高
```

## 四、ETL流程

### 4.1 ETL概述

```
ETL流程
        │
        ├─ E（Extract）：抽取数据
        │   ├─ 全量抽取：抽取所有数据
        │   ├─ 增量抽取：抽取新增数据
        │   └─ CDC抽取：变更数据捕获
        │
        ├─ T（Transform）：转换数据
        │   ├─ 数据清洗：去除脏数据
        │   ├─ 数据转换：格式转换
        │   ├─ 数据映射：字段映射
        │   └─ 数据聚合：汇总数据
        │
        └─ L（Load）：加载数据
            ├─ 全量加载：覆盖加载
            ├─ 增量加载：追加加载
            └─ 缓慢变化维处理
```

### 4.2 缓慢变化维（SCD）

```
缓慢变化维处理策略
        │
        ├─ SCD Type 1：覆盖旧值
        │   ├─ 适用场景：不关心历史变化
        │   └─ 示例：产品价格更新
        │
        ├─ SCD Type 2：添加新记录
        │   ├─ 适用场景：需要保留历史变化
        │   ├─ 添加开始日期和结束日期
        │   └─ 示例：客户地址变更
        │
        └─ SCD Type 3：添加新字段
            ├─ 适用场景：只需保留最近几次变化
            └─ 添加历史字段
```

### 4.3 ETL工具

```
ETL工具
        │
        ├─ 开源工具
        │   ├─ Apache NiFi：数据流自动化
        │   ├─ Apache Airflow：工作流调度
        │   ├─ DataX：阿里巴巴开源数据同步工具
        │   └─ Sqoop：Hadoop数据迁移工具
        │
        ├─ 商业工具
        │   ├─ Informatica PowerCenter
        │   ├─ IBM DataStage
        │   ├─ Microsoft SSIS
        │   └─ Oracle Data Integrator
        │
        └─ 云服务
            ├─ AWS Glue：托管ETL服务
            ├─ Azure Data Factory：Azure数据工厂
            └─ Google Cloud Dataflow：Google数据流
```

## 六、数据仓库优化

### 6.1 查询优化

```
查询优化策略
        │
        ├─ 预计算：提前计算汇总数据
        ├─ 索引优化：创建合适的索引
        ├─ 分区策略：按时间或业务维度分区
        ├─ 缓存机制：缓存常用查询结果
        └─ 数据压缩：减少存储和I/O
```

### 6.2 存储优化

```
存储优化策略
        │
        ├─ 数据压缩：列式存储和压缩
        ├─ 冷热分离：冷热数据分开存储
        ├─ 存储格式：选择合适的存储格式（Parquet、ORC）
        └─ 生命周期管理：自动管理数据生命周期
```

## 总结

数据仓库设计是一个复杂的过程，需要结合业务需求、数据特点和技术选型来决定。通过合理的数据建模、ETL流程设计和性能优化，可以构建高效、可靠的数据仓库系统。
