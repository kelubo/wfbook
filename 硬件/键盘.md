# 键盘

本文件整理 Arduino `Keyboard` 库（用于 Leonardo、Micro、Pro Micro、Due、Zero、MKR 等支持 USB HID 功能的板卡）中可用的键值宏。这些键值基于 USB HID Usage Tables，可作为 `Keyboard.write()`、`Keyboard.press()`、`Keyboard.release()` 等函数的参数，用于模拟键盘按键。

> 数据来源：[Arduino 官方文档 - Keyboard Modifiers and Special Keys](https://docs.arduino.cc/language-reference/en/functions/usb/Keyboard/keyboardModifiers)（最后修订 2024-05-15）。键值与 `Keyboard.h` 头文件中 `#define` 一致。

## 键值表

### 键盘修饰键 (Keyboard modifiers)

|-----------------|--------------|--------------|-------------------------------|

| KEY_LEFT_SHIFT  | 129          | 0x81         | 左 Shift 键                   |

## 使用示例

```cpp
#include <Keyboard.h>

void setup() {
  // 初始化虚拟键盘
  Keyboard.begin();
  delay(1000);  // 留出时间给宿主设备识别

  // 模拟快捷键 Ctrl + Alt + Delete
  Keyboard.press(KEY_LEFT_CTRL);
  Keyboard.press(KEY_LEFT_ALT);
  Keyboard.press(KEY_DELETE);
  delay(100);
  Keyboard.releaseAll();

  // 输出字符串
  Keyboard.println("Hello, world!");

  // 模拟单个功能键
  Keyboard.press(KEY_F5);
  delay(100);
  Keyboard.release(KEY_F5);
}

void loop() {
  // 空循环
}
```

## 注意事项

1. **支持板卡**：仅原生支持 USB HID 功能的 Arduino 板卡（如基于 ATmega32U4 的 Leonardo、Micro、Pro Micro，基于 SAMD 的 Zero、Due、MKR 系列）才能使用 `Keyboard` 库。Uno、Mega 等板卡需刷写特定固件才能模拟 HID 设备，普通方法无法直接使用。
2. **驱动占用**：在烧录程序时，`Keyboard.begin()` 会立即向 PC 发送按键事件，可能干扰 IDE 自身的输入。建议在 `setup()` 开头加入 1~5 秒延时，便于再次烧录。
3. **释放按键**：模拟按键时必须配套调用 `Keyboard.release()` 或 `Keyboard.releaseAll()`，否则对应按键会一直保持"按下"状态。
4. **字符与键值**：可直接使用 ASCII 字符（如 `Keyboard.write('A')`），库会自动处理 Shift 组合；只有非可打印字符才需要使用本表中的键值宏。
5. **数据来源**：本表键值与 Arduino IDE 自带 `Keyboard.h` 头文件中 `#define` 的值一致，对应 USB HID Usage Tables 中键盘部分的 Usage ID（0x80 以上）。
