# 网络监控

## 概述

网络监控是对网络设备、链路和应用进行实时监测和管理，确保网络稳定运行。

## 监控层次

```
┌─────────────────────────────────────────┐
│              应用层监控                 │
│    HTTP响应时间、API可用性             │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│              网络层监控                 │
│    带宽利用率、延迟、丢包率             │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│              设备层监控                 │
│    CPU使用率、内存、接口状态           │
└─────────────────────────────────────────┘
```

## 监控工具

### Prometheus

#### 安装

```bash
# 使用Docker安装
docker run -d -p 9090:9090 --name prometheus prom/prometheus

# 配置文件
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'node_exporter'
    static_configs:
      - targets: ['localhost:9100']
```

#### 导出器

```bash
# Node Exporter（主机监控）
docker run -d -p 9100:9100 --name node_exporter \
    --net="host" --pid="host" \
    -v "/:/host:ro,rslave" \
    quay.io/prometheus/node-exporter:latest

# SNMP Exporter（网络设备监控）
docker run -d -p 9116:9116 --name snmp_exporter \
    -v /path/to/snmp.yml:/etc/snmp_exporter/snmp.yml \
    prom/snmp-exporter
```

#### 查询语言

```promql
# CPU使用率
100 - (avg by (instance) (irate(node_cpu_seconds_total{mode="idle"}[1m])) * 100)

# 内存使用率
100 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes * 100)

# 网络带宽
rate(node_network_receive_bytes_total[1m])

# 磁盘使用率
100 - (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"} * 100)
```

### Grafana

#### 安装

```bash
# 使用Docker安装
docker run -d -p 3000:3000 --name grafana grafana/grafana

# 登录
# 访问 http://localhost:3000
# 默认用户名/密码：admin/admin
```

#### 配置数据源

```
1. 登录Grafana
2. 点击左侧菜单 "Configuration" → "Data Sources"
3. 点击 "Add data source"
4. 选择 "Prometheus"
5. 输入URL：http://prometheus:9090
6. 点击 "Save & Test"
```

#### 导入仪表盘

```
1. 点击左侧菜单 "Dashboards" → "Import"
2. 输入仪表盘ID：1860（Node Exporter Full）
3. 选择Prometheus数据源
4. 点击 "Import"
```

### Zabbix

#### 安装

```bash
# 使用Docker安装
docker run -d --name zabbix-server -p 10051:10051 zabbix/zabbix-server-mysql

docker run -d --name zabbix-web -p 8080:8080 zabbix/zabbix-web-nginx-mysql

# 登录
# 访问 http://localhost:8080
# 默认用户名/密码：Admin/zabbix
```

#### 添加主机

```
1. 登录Zabbix
2. 点击 "Configuration" → "Hosts"
3. 点击 "Create host"
4. 输入主机名称和IP地址
5. 在 "Templates" 标签页添加模板（如 Template OS Linux）
6. 点击 "Add"
```

#### 创建监控项

```
1. 点击 "Configuration" → "Hosts"
2. 选择主机，点击 "Items"
3. 点击 "Create item"
4. 输入名称、键值、类型等信息
5. 点击 "Add"
```

### Nagios

#### 安装

```bash
# 安装Nagios Core
sudo apt-get install nagios3 nagios-plugins

# 配置文件目录
/etc/nagios3/
  ├── nagios.cfg          # 主配置文件
  ├── conf.d/             # 配置目录
  └── objects/            # 对象定义
```

#### 配置监控主机

```
# /etc/nagios3/conf.d/hosts.cfg
define host {
    host_name           server1
    alias               Server 1
    address             192.168.1.10
    check_command       check-host-alive
    max_check_attempts  3
    check_interval      5
    retry_interval      1
    check_period        24x7
    notification_period 24x7
}
```

#### 配置服务监控

```
# /etc/nagios3/conf.d/services.cfg
define service {
    host_name           server1
    service_description HTTP
    check_command       check_http
    max_check_attempts  3
    check_interval      5
    retry_interval      1
    check_period        24x7
    notification_period 24x7
}
```

## 网络流量分析

### Wireshark

```bash
# 安装Wireshark
sudo apt-get install wireshark

# 启动Wireshark
wireshark

# 常用过滤规则
ip.addr == 192.168.1.1        # 过滤特定IP
tcp.port == 80                 # 过滤特定端口
http.request.method == "GET"  # 过滤HTTP GET请求
icmp                           # 过滤ICMP协议
```

### tcpdump

```bash
# 捕获所有流量
tcpdump -i eth0

# 捕获特定IP的流量
tcpdump host 192.168.1.1

# 捕获特定端口的流量
tcpdump port 80

# 保存到文件
tcpdump -i eth0 -w capture.pcap

# 读取文件
tcpdump -r capture.pcap
```

## 告警系统

### Alertmanager

```yaml
# alertmanager.yml
global:
  smtp_smarthost: 'smtp.example.com:587'
  smtp_from: 'alertmanager@example.com'

route:
  group_by: ['alertname']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  receiver: 'email'

receivers:
- name: 'email'
  email_configs:
  - to: 'admin@example.com'
```

### Prometheus告警规则

```yaml
# alert_rules.yml
groups:
- name: host_alerts
  rules:
  - alert: HighCPUUsage
    expr: 100 - (avg by (instance) (irate(node_cpu_seconds_total{mode="idle"}[1m])) * 100) > 90
    for: 5m
    labels:
      severity: critical
    annotations:
      summary: "High CPU usage on {{ $labels.instance }}"
      description: "CPU usage is {{ $value }}%"
```

## 网络监控最佳实践

### 监控指标选择

- **关键指标**：CPU、内存、磁盘、网络
- **业务指标**：响应时间、可用性、吞吐量
- **预测指标**：趋势分析、容量规划

### 告警策略

- **分级告警**：紧急、重要、一般、警告
- **告警抑制**：避免告警风暴
- **告警升级**：未处理时升级通知

### 监控可视化

- **仪表盘**：实时状态展示
- **趋势图**：历史数据分析
- **拓扑图**：网络结构展示

## 总结

网络监控是保障网络稳定运行的关键，通过综合使用Prometheus、Grafana、Zabbix等工具，可以实现全面的网络监控和告警。
