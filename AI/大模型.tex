\part{入门篇：什么是大模型}

\chapter{从零开始认识大模型}

\section{什么是"大模型"}

"大模型"是"大规模预训练模型"的简称，英文叫 \textbf{Large Language Model}，缩写 \textbf{LLM}。简单来说，就是一种"读过海量书籍、能说人话、还会推理"的人工智能程序。

这里的"大"有两层含义：
\begin{itemize}
  \item \textbf{训练数据大}：用了几千亿甚至几万亿字的文本（书、网页、新闻、代码等）来"喂"它。
  \item \textbf{模型参数大}：模型内部有几十亿、几百亿甚至上千亿个"旋钮"（参数），需要巨大的算力来调整。
\end{itemize}

举个例子，一个普通的小模型可能只有几百万个参数，相当于一个"小学生"；而 GPT-4、DeepSeek 这类大模型有上千亿个参数，相当于一个"博士后"。

\section{大模型能做什么}

大模型最擅长的就是\textbf{理解和生成自然语言}。具体能做的事包括：

\begin{enumerate}
  \item \textbf{问答对话}：你可以问它任何问题，它会用流畅的语言回答你。
  \item \textbf{文章写作}：写邮件、写报告、写小说、写诗歌都不在话下。
  \item \textbf{翻译}：中英日法德等几十种语言互译。
  \item \textbf{代码生成}：根据你的描述写程序，或者帮你找 bug。
  \item \textbf{总结归纳}：把一篇长文压缩成几句话的摘要。
  \item \textbf{逻辑推理}：数学题、逻辑题、规划任务也能做。
  \item \textbf{角色扮演}：让它扮演老师、医生、律师等特定身份回答问题。
\end{enumerate}

\section{大模型不能做什么}

虽然大模型很强，但它也有明显的短板：

\begin{itemize}
  \item \textbf{不能联网实时查询}：它的知识有截止日期，问它"今天天气如何"是没用的（除非专门联网）。
  \item \textbf{可能"一本正经地胡说"}：这种现象叫"幻觉"（Hallucination），它会编造看起来合理但实际是假的信息。
  \item \textbf{不能执行操作}：本身不能发邮件、下订单、控制系统（除非配合专门的工具）。
  \item \textbf{不能保证数学计算百分百正确}：复杂的算术题也可能算错。
  \item \textbf{不能突破训练数据}：训练数据里没有的内容它也无从知晓。
\end{itemize}

\section{常见名词解释}

\begin{description}
  \item[LLM] Large Language Model，大语言模型。
  \item[Token] 模型处理文本的最小单位，约等于 1 个汉字或半个英文单词。
  \item[参数] 模型内部的"旋钮"，参数越多通常越聪明，但也越慢、越贵。
  \item[上下文] 一次对话中模型能"记住"的内容长度，比如 4K、32K、128K 等。
  \item[Prompt] 你发给模型的提示词，也就是你的问题或指令。
  \item[微调] 在已经训练好的模型基础上，用专门数据再训练，让它更擅长某类任务。
  \item[开源] 模型权重公开，可以免费下载使用；闭源则只能通过 API 调用。
  \item[幻觉] 模型生成看似合理但实际错误的内容。
\end{description}

\section{Transformer 是什么}

几乎所有现代大模型都基于一个叫做 \textbf{Transformer} 的架构。2017 年，Google 的一篇论文《Attention Is All You Need》提出了这个架构，从此开启了 AI 的新时代。

简单一句话理解：

\begin{quote}
Transformer 是一种\textbf{让计算机"同时关注一句话里所有词"}的方法，它通过"注意力机制"来理解上下文关系。
\end{quote}

举个例子，当你说"小明把苹果给了小红，因为她很喜欢吃"时，模型需要知道"她"指的是"小红"。Transformer 的"注意力机制"能自动找出这种指代关系。

为什么 Transformer 这么厉害？
\begin{itemize}
  \item \textbf{并行计算}：比之前的 RNN 快得多，训练时间大幅缩短
  \item \textbf{长距离依赖}：能同时处理很长的文本
  \item \textbf{可扩展性强}：堆叠更多层就能得到更大的模型
\end{itemize}

简单说，Transformer 就是现代大模型的"基石"。GPT、LLaMA、Qwen 等所有主流模型，都是 Transformer 的"变体"或"后代"。

\chapter{大模型是怎么"炼"出来的}

\section{基本流程}

训练一个大模型，简单来说分为三步：

\begin{enumerate}
  \item \textbf{预训练（Pre-training）}：用海量文本训练，让模型学会"语言的规律"和"世界知识"。这一步非常烧钱，可能要几百万到几千万美元。
  \item \textbf{指令微调（SFT, Supervised Fine-Tuning）}：用人工编写的"问题-答案"对训练，让模型学会"听懂人话、按照指令回答"。
  \item \textbf{人类反馈强化学习（RLHF）}：让人类对模型的多个回答打分，训练模型"说人话、说对话"。
\end{enumerate}

\section{训练数据从哪里来}

大模型的"知识"全部来自训练数据。那么这些数据从哪里来呢？以下是典型的大模型训练数据构成：

\begin{itemize}
  \item \textbf{网页数据}（占比最大，约 60\%-80\%）
  \begin{itemize}
    \item Common Crawl：开源的全球网页抓取项目
    \item 维基百科：高质量的百科知识
    \item Reddit、知乎等社区内容
  \end{itemize}
  \item \textbf{书籍}（约 10\%-20\%）
  \begin{itemize}
    \item 小说、教科书、专业书籍
    \item Books3、Project Gutenberg 等数据集
  \end{itemize}
  \item \textbf{代码}（约 5\%-15\%）
  \begin{itemize}
    \item GitHub 上的开源代码
    \item Stack Overflow 等技术问答
  \end{itemize}
  \item \textbf{学术论文}（约 2\%-5\%）
  \begin{itemize}
    \item arXiv、PubMed 等学术资源
  \end{itemize}
  \item \textbf{对话与指令数据}（专门为微调准备）
  \begin{itemize}
    \item ShareGPT、Alpaca 等
  \end{itemize}
\end{itemize}

数据清洗非常重要：原始数据中包含大量垃圾信息、错误内容、重复内容，需要经过严格的过滤、去重、脱敏处理后，才能用于训练。

\textbf{数据配比的关键原则}：高质量数据 > 低质量数据。研究者发现，把维基百科和优质书籍多放一些，比多抓一些网页效果更好。

\section{预训练与微调}

打个比方：
\begin{itemize}
  \item \textbf{预训练}就像一个人读完了整个图书馆的书，知识渊博但不会回答问题。
  \item \textbf{指令微调}就是请一位老师教它"别人问什么该怎么答"。
  \item \textbf{强化学习}就是让它不断练习，根据老师的评分改进。
\end{itemize}

\section{什么是"参数"}

"参数"是模型内部的数字，可以理解为模型"思考问题时使用的旋钮"。模型越大，参数越多，能学的知识越复杂。

参数单位：
\begin{itemize}
  \item 1B = 10亿（1 Billion）
  \item 7B = 70亿
  \item 70B = 700亿
  \item 671B = 6710亿
\end{itemize}

一般规律：
\begin{itemize}
  \item 1.5B 左右的模型：手机都能跑，适合简单任务
  \item 7B-14B：需要普通显卡（8GB-16GB 显存）
  \item 70B 以上：需要专业显卡或多卡集群
\end{itemize}

\section{为什么叫"涌现能力"}

"涌现"（Emergence）是一个神奇的现象：当模型小到一定程度时，它的能力增长很慢；但当模型大到某个临界点后，会\textbf{突然出现}一些新能力，比如推理、翻译、写代码等。

就像蚂蚁单个很笨，但一群蚂蚁能建造复杂的蚁巢——大模型的智能是"涌现"出来的，而不是简单堆参数。

\section{各家族特点对比}

\begin{itemize}
  \item \textbf{闭源商业派}（GPT、Claude、Gemini）：质量最高，但按调用付费
  \item \textbf{开源实力派}（DeepSeek、Qwen、LLaMA）：免费可商用，质量也很好
  \item \textbf{垂直专精派}（Phi、Granite）：参数小但效率高，适合特定场景
  \item \textbf{多模态派}（GPT-4o、Gemini、Qwen-VL）：能看图、看视频
\end{itemize}

\textbf{如何看懂这张图}：
\begin{itemize}
  \item 每个"父模型"都有一群"子模型"（衍生模型）
  \item 衍生模型通常是用父模型微调而来，专精某个方向
  \item 比如 LLaMA 是"开源之父"，衍生出 Vicuna（聊天）、Wizard（指令）、Dolphin（无审查）等几十种模型
\end{itemize}

\textbf{2026 年的趋势}：
\begin{itemize}
  \item \textbf{推理模型崛起}：OpenAI o1/o3、DeepSeek-R1、QwQ 等专门强化"思考"能力
  \item \textbf{开源追赶闭源}：DeepSeek-V3/R1 在多项指标上已经接近 GPT-4
  \item \textbf{多模态普及}：能看图、看视频的模型成为标配
  \item \textbf{端侧部署}：1.5B-7B 模型在手机、笔记本上流畅运行
\end{itemize}

\chapter{如何选择适合自己的模型}

\section{看参数规模}

\begin{itemize}
  \item \textbf{1B-3B}：手机/树莓派可跑，适合嵌入式设备
  \item \textbf{7B-14B}：消费级显卡可跑，平衡性能与速度
  \item \textbf{32B-72B}：需要专业显卡，效果接近商业大模型
  \item \textbf{100B+}：需要多卡或云服务，效果最强
\end{itemize}

\section{看使用场景}

\begin{itemize}
  \item 日常聊天、写作：7B-14B 足够
  \item 编程辅助：建议 14B 以上的代码专用模型（如 Qwen2.5-Coder）
  \item 数学推理：DeepSeek-R1、QwQ 等推理模型
  \item 多语言翻译：Qwen、Llama 等多语言支持好的模型
\end{itemize}

\section{看是否开源}

\begin{itemize}
  \item \textbf{开源}（DeepSeek、Qwen、LLaMA）：可下载、可商用、可定制
  \item \textbf{闭源}（GPT、Claude）：质量高但只能调用 API
\end{itemize}

\section{看上下文长度}

\begin{itemize}
  \item 4K-8K：日常对话
  \item 32K-128K：长文档分析
  \item 200K+：超长文档、整本书分析
\end{itemize}

\section{看是否支持工具调用}

工具调用（Function Calling / Tool Use）让模型能"使用工具"（查天气、查数据库、执行代码等）。如果要做 AI Agent，就需要选择支持工具调用的模型。

\chapter{自己训练大模型}

前面我们都是"用"别人的大模型。这一章，我们来看看\textbf{怎么自己训练}一个大模型。

听起来很难？其实普通人完全可以做到。不需要从零开始（那需要几百万美元），而是在现有模型的基础上"再训练"一下。

\section{训练大模型的四种方式}

根据自己的目标，从易到难有 4 种"训练"方式：

\subsection{方式 1：提示工程（Prompt Engineering）}

\textbf{难度}：⭐☆☆☆☆

这是最简单的方式——\textbf{不改模型，只改输入}。

\begin{verbatim}
已有大模型 → 设计更好的 Prompt → 输出质量提升
\end{verbatim}

\subsection{方式 2：检索增强生成（RAG）}

\textbf{难度}：⭐⭐☆☆☆

给模型"外挂"一个知识库，回答问题时先去查资料。

\begin{verbatim}
用户问题 → 检索相关文档 → 把文档给模型 → 模型参考后回答
\end{verbatim}

\textbf{适合}：让 AI 回答专业领域问题、企业知识库

\subsection{方式 3：微调（Fine-tuning）}

\textbf{难度}：⭐⭐⭐⭐☆

用专有数据继续训练已有模型。

\begin{verbatim}
开源大模型 → 用自己的数据训练 → 得到定制模型
\end{verbatim}

\textbf{适合}：垂直领域（医疗、法律、客服）

\subsection{方式 4：从头预训练（Pre-training from scratch）}

\textbf{难度}：⭐⭐⭐⭐⭐

从零开始训练一个新模型。

\begin{verbatim}
随机初始化 → 海量数据训练 → 得到基础模型
\end{verbatim}

\textbf{适合}：科研机构、大公司

\section{各方式的代价对比}

\begin{tabular}{|l|l|l|l|l|}
\hline
\textbf{方式} & \textbf{数据量} & \textbf{算力} & \textbf{耗时} & \textbf{金钱} \\
\hline
Prompt & 0 & 0 & 几分钟 & 0 \\
RAG & 几 MB & CPU 即可 & 几小时 & 0 \\
微调 & 1万-100万条 & 1-8 张 GPU & 几天 & 几百到几万 \\
预训练 & 几 TB & 数百张 GPU & 数月 & 几百万美元 \\
\hline
\end{tabular}

\section{最适合普通人的方案：LoRA 微调}

\textbf{LoRA}（Low-Rank Adaptation）是一种轻量级微调技术。

\textbf{为什么推荐 LoRA}：
\begin{itemize}
  \item 不需要从头训练
  \item 一张消费级显卡（如 RTX 4090）就能跑
  \item 训练一个"专属模型"只要几百块
  \item 学会后可以做很多实际项目
\end{itemize}

\textbf{LoRA 的原理}：

原模型参数：冻结不动

新增 LoRA 参数：可训练（只占原模型的 0.1\%-1\%）

效果：用 1\% 的算力，达到接近全量微调的效果

\section{动手做一个小模型}

下面是一个\textbf{真正能跑通}的端到端项目。

\subsection{项目目标}

训练一个"古诗风格转换器"：让大模型学会"用李白的风格写诗"。

\subsection{步骤 1：环境准备}

\begin{verbatim}

\textbf{关键指标}：
\begin{itemize}
  \item \textbf{Loss（损失）}：越低越好，表示模型学得越准
  \item \textbf{学习率}：控制训练速度
  \item \textbf{困惑度（Perplexity）}：评价语言模型好坏
\end{itemize}

\chapter{本地部署大模型}

\section{什么是本地部署}

"本地部署"就是把大模型下载到你自己的电脑或服务器上运行，\textbf{不需要联网}，所有数据都在本地处理。

优点：
\begin{itemize}
  \item \textbf{隐私性好}：数据不出本地，适合处理敏感信息
  \item \textbf{免费}：一次下载，永久使用
  \item \textbf{可定制}：可以根据需要修改
\end{itemize}

缺点：
\begin{itemize}
  \item \textbf{硬件要求高}：大模型需要强大的显卡（GPU）
  \item \textbf{配置复杂}：需要一定的技术基础
  \item \textbf{更新慢}：新版模型需要自己下载
\end{itemize}

\section{常用工具：Ollama、LM Studio、Jan}

\textbf{Ollama}
\begin{itemize}
  \item 最流行的本地大模型运行工具
  \item 命令行操作，简单易用
  \item 跨平台：Windows、macOS、Linux
  \item 官网：\texttt{https://ollama.com}
\end{itemize}

\textbf{LM Studio}
\begin{itemize}
  \item 图形界面，适合新手
  \item 内置模型市场
  \item 可视化参数调整
\end{itemize}

\textbf{Jan}
\begin{itemize}
  \item 开源、跨平台
  \item 界面美观
  \item 支持本地 API 服务器
\end{itemize}

\section{硬件要求}

\begin{itemize}
  \item \textbf{1.5B-3B 模型}：8GB 内存即可（无独显也能跑，慢一点）
  \item \textbf{7B 模型}：建议 8GB 显存的显卡（如 RTX 3070）
  \item \textbf{14B 模型}：建议 12GB 显存的显卡（如 RTX 4070）
  \item \textbf{32B 模型}：建议 24GB 显存的显卡（如 RTX 4090）
  \item \textbf{70B+ 模型}：需要多张专业显卡（如 A100、H100）
\end{itemize}

\section{快速上手}

以 Ollama 为例，三步搞定：

  \item \textbf{下载模型}：在终端运行

ollama pull qwen2.5:7b
\end{verbatim}
  \item \textbf{开始对话}：

\end{verbatim}
\end{enumerate}

\section{常见问题排查}

\textbf{问题 1：模型下载太慢}

模型文件通常有几个 G 到几十个 G，从国外服务器下载很慢。

\textbf{解决方案}：
\begin{itemize}
  \item 使用国内镜像（如 \texttt{https://hf-mirror.com}）
  \item 设置环境变量：
\begin{verbatim}
# Linux/macOS
export HF_ENDPOINT=https://hf-mirror.com
# Windows PowerShell
$env:HF_ENDPOINT="https://hf-mirror.com"
\end{verbatim}
  \item 使用迅雷、IDM 等下载工具手动下载
\end{itemize}

\textbf{问题 2：显存不足（OOM）}

运行时报错 \texttt{CUDA out of memory}。

\textbf{解决方案}：
\begin{itemize}
  \item 换用更小的模型（如 7B 换成 1.5B 或 3B）
  \item 使用量化版本（如 Q4\_K\_M 量化版，体积小一半但效果接近）
  \item 关闭其他占用显存的应用
  \item 升级显卡（终极方案）
\end{itemize}

\textbf{量化版本参考}：
\begin{itemize}
  \item Q2\_K：体积最小，质量最差
  \item Q4\_K\_M：平衡选择（推荐）
  \item Q5\_K\_M、Q6\_K：质量更高
  \item Q8\_0：几乎无损
  \item F16/F32：原始精度
\end{itemize}

\textbf{问题 3：模型运行卡顿}

CPU 模式下非常慢。

\textbf{解决方案}：
\begin{itemize}
  \item 优先使用 GPU 模式（NVIDIA 显卡）
  \item Apple Silicon（M1/M2/M3/M4）使用 Metal 加速
  \item CPU 模式只推荐运行 1.5B 以下的模型
\end{itemize}

\textbf{问题 4：API 调用报错}

常见错误：401（密钥错误）、429（限流）、500（服务器问题）。

\textbf{解决方案}：
\begin{itemize}
  \item 检查 API 密钥是否正确
  \item 降低调用频率
  \item 等待几分钟后重试
  \item 联系 API 提供商
\end{itemize}

\textbf{问题 5：模型回答不靠谱}

明明问得清楚，模型却胡说八道。

\textbf{解决方案}：
\begin{itemize}
  \item 换用更大的模型
  \item 改进 Prompt（详见下章）
  \item 提供更多上下文
  \item 设置 \texttt{temperature=0} 让回答更稳定
\end{itemize}

\section{Prompt 工程基础}

"Prompt"就是你和模型说的话。**好的 Prompt 能让模型输出质量提升 10 倍**。这就是"Prompt 工程"。

\subsection{技巧 1：明确角色}

给模型一个"身份"，效果会好很多。

\textbf{差 Prompt}：
\begin{verbatim}
写一篇关于环保的文章。
\end{verbatim}

\textbf{好 Prompt}：
\begin{verbatim}
你是一位有 20 年经验的环保科普作家，请写一篇
800 字左右的文章，主题是"普通家庭如何减少碳足迹"，
要求语言通俗易懂，适合中学生阅读。
\end{verbatim}

\subsection{技巧 2：提供示例}

如果想让模型按特定格式输出，给一两个示例。

\textbf{示例}：
\begin{verbatim}
请将以下商品评论分类为：正面、中性、负面。

示例 1：
评论：这个东西质量太差了！
分类：负面

示例 2：
评论：还行吧，没什么特别的感觉。
分类：中性

现在请分类：
评论：质量很好，物流也快，下次还会买！
分类：
\end{verbatim}

\subsection{技巧 3：分步思考}

对于复杂问题，让模型"一步步想"。

\textbf{示例}：
\begin{verbatim}
某商店袜子 10 元一双，买 5 双送 1 双。
我有 100 元，最多能拿到多少双袜子？
请一步步分析。
\end{verbatim}

\subsection{技巧 4：限制输出}

明确告诉模型"不要做什么"。

\textbf{示例}：
\begin{verbatim}
请用 200 字以内介绍相对论。要求：
- 不要使用专业术语
- 用日常生活的例子
- 不要超过 3 个段落
\end{verbatim}

\subsection{技巧 5：Few-shot 提示}

提供多个示例，让模型"照葫芦画瓢"。

\textbf{示例（文本分类）}：
\begin{verbatim}
请把以下句子按情感分类。

例：
"今天天气真好" → 积极
"我肚子很饿" → 中性
"我考试不及格" → 消极

现在分类：
"我升职加薪了" →
\end{verbatim}

\subsection{常用 Prompt 模板}

\begin{itemize}
  \item \textbf{角色扮演}："你是一位[职业]，请[任务]"
  \item \textbf{步骤拆解}："请分步骤分析[问题]"
  \item \textbf{格式约束}："请用[格式]输出"
  \item \textbf{反向提问}："在回答前，先问我 3 个关键问题"
  \item \textbf{自我检查}："回答后，请检查是否正确"
\end{itemize}

\section{AI Agent 简介}

如果说大模型是一个"聪明的学生"，那 AI Agent 就是一个"能自主干活的助手"。

\subsection{什么是 AI Agent}

\textbf{AI Agent}（人工智能智能体）= 大模型 + 工具 + 记忆 + 规划

它不只是"回答问题"，而是可以：
\begin{itemize}
  \item 自主规划任务步骤
  \item 调用外部工具（搜索、计算、发邮件、写代码）
  \item 观察执行结果，调整下一步行动
  \item 完成多步骤复杂任务
\end{itemize}

\subsection{AI Agent vs 普通对话}

\begin{itemize}
  \item \textbf{普通对话}：你问，它答，结束
  \item \textbf{AI Agent}：你给一个目标，它自己规划、自己执行、最后给你结果
\end{itemize}

\textbf{示例}：

普通对话：
\begin{verbatim}
你：明天的天气怎么样？
AI：抱歉，我无法访问实时数据。
\end{verbatim}

AI Agent：
\begin{verbatim}
你：明天上海会下雨吗？如果会，提醒我带伞。
Agent：好的，我先查询天气...
  [调用天气 API]
  [结果：上海明天有雨]
Agent：明天上海有雨，我已经：
  1. 查询了天气预报
  2. 在你的日历中添加了"带伞"提醒
  3. 给你的手机发了推送通知
\end{verbatim}

\subsection{常见 AI Agent 框架}

\begin{itemize}
  \item \textbf{LangChain}：最流行的 Python Agent 框架
  \item \textbf{LlamaIndex}：专注 RAG 和数据连接
  \item \textbf{AutoGen}（微软）：多 Agent 协作
  \item \textbf{CrewAI}：角色扮演式多 Agent
  \item \textbf{OpenAI Agents SDK}：OpenAI 官方
  \item \textbf{Anthropic MCP}：模型上下文协议标准
\end{itemize}

\subsection{应用场景}

\begin{itemize}
  \item \textbf{编程助手}：Claude Code、Cursor、Copilot Agent
  \item \textbf{自动化办公}：自动处理 Excel、写周报、整理邮件
  \item \textbf{信息检索}：自动搜索、整理、总结
  \item \textbf{个人助理}：日程管理、订餐、订票
  \item \textbf{科研助手}：读论文、做文献综述、设计实验
\end{itemize}

\subsection{简单 Agent 示例}

使用 LangChain 创建一个会查天气的 Agent：

\begin{verbatim}
from langchain.agents import Agent
from langchain.tools import tool

@tool
def get_weather(city: str) -> str:
    """查询某城市天气"""
    # 实际中调用天气 API
    return f"{city}：晴，25 度"

agent = Agent(tools=[get_weather])
result = agent.run("上海明天会下雨吗？")
\end{verbatim}

这就是 AI Agent 的雏形：让大模型"使用工具"，完成更复杂的任务。

\chapter{深度学习模型}

\section{DeepSeek 系列}

\subsection{deepseek-r1}

DeepSeek's first-generation of reasoning models with comparable performance to OpenAI-o1, including six dense models distilled from DeepSeek-R1 based on Llama and Qwen.
DeepSeek 的第一代推理模型，性能可与 OpenAI-o1 相媲美，包括从 DeepSeek-R1 中提炼出来的基于 Llama 和 Qwen 的 6 个密集模型。

参数规模：1.5b、7b、8b、14b、32b、70b、671b

\subsection{DeepSeek R1 Distill}

DeepSeek R1 Distill 是 DeepSeek 公司基于 R1 系列模型进行蒸馏得到的轻量级模型，保留了原始模型的核心能力，同时显著减小了模型体积，提高了推理速度。适用于资源受限的场景。

\subsection{deepseek-v3}

A strong Mixture-of-Experts (MoE) language model with 671B total parameters with 37B activated for each token.
一个强大的专家混合（MoE）语言模型，总共有 671B 个参数，每个标记激活了 37B。

参数规模：671b

\subsection{deepseek-coder}

DeepSeek Coder is a capable coding model trained on two trillion code and natural language tokens.
DeepSeek Coder 是一个功能强大的编码模型，使用 2 万亿个代码和自然语言标记进行训练。

参数规模：1.3b、6.7b、33b

\subsection{deepseek-coder-v2}

An open-source Mixture-of-Experts code language model that achieves performance comparable to GPT4-Turbo in code-specific tasks.
一种开源 Mixture-of-Experts 代码语言模型，可在特定于代码的任务中实现与 GPT4-Turbo 相当的性能。

参数规模：16b、236b

\subsection{deepscaler}

A fine-tuned version of Deepseek-R1-Distilled-Qwen-1.5B that surpasses the performance of OpenAI's o1-preview with just 1.5B parameters on popular math evaluations.
Deepseek-R1-Distilled-Qwen-1.5B 的微调版本，在流行的数学评估中仅具有 1.5B 参数，其性能超过了 OpenAI 的 o1-preview。

参数规模：1.5b

\section{LLaMA 系列}

\subsection{llama3.3}

New state of the art 70B model. Llama 3.3 70B offers similar performance compared to the Llama 3.1 405B model.
最先进的 70B 型号。与 Llama 3.3 70B 型号相比，Llama 3.1 405B 提供相似的性能。

参数规模：70b，支持工具调用

\subsection{llama3.2}

Meta's Llama 3.2 goes small with 1B and 3B models.
